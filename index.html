    agni

    Simple and intuitive MVC web framework for node.js.

    Overview

    Agni is a web framework built on top of Express, based on the principle of convention over configuration.

    Its purpose is making access to the various components easier, so that you can forget about routes and paths and focus on what your application does.

    It does not include any database abstraction system, but leaves this choice to the developer.

    Quick start

    1. Install agni globally: npm install agni -g.

    2. Create an application called welcome: agni create welcome

    3. Enter the newly created directory: cd welcome.

    4. Install all dependencies: npm install.

    5. Run the application: nmp start.

    6. Point your browser to localhost:3000.

    7. If you see the welcome message, it works!

    Architecture

    Agni is loosely based on the MVC development pattern.

    The three main application components are:

    • the Model, which is the data access layer;

    • the View, which is the part of the application that is "visible" to the user;

    • the Controller, which acts as an intermediary between the other two components, taking data from the Model and sending it to the View.

    A fourth component is also available, that is the Service layer. The purpose of this layer is encapsulating business logic in a high level interface available to the controller, freeing the latter from overload and duplication.

    Of course, you are free to use only the components that you need. Agni does not oblige you to use Models and Services if you do not need them.

    Directory structure

    The root application directory is app.

    Inside app, you have controllers, models, services and views, where you can store the modules belonging to the corresponding components.

    In config there is a configuration file config.js and an initialization script init.js.

    In public you can put all your static resources like images, javascript files, style sheet and so on.

    public is also the default favicon location.

    Routing

    There is no explicit route definition in Agni. The routing mechanism is based on controller paths, function names and (optionally) HTTP methods.

    Controllers

    The Controller as a router

    Agni uses a common routing system, similar to what is used in CodeIgniter and other frameworks.

    If you split the requested URL in segments:

    mywebsite.com/module/function/arg1/arg2

    1. The first URL segment is the module that should be required.

    2. The second segment is the function that should be called.

    3. The third, and any other segments, are the arguments that should be passed to the function.

    So, for example, a request to mywebsite.com/article/show/173, would require the article module from the controllers folder, and execute the exported function show with 173 as parameter.

    More or less equivalent to doing:

    var article = require('./app/controllers/article');
article.show(173);

    If you omit the method name, it will look for a function called nil and execute it.

    For example, a request to mywebsite.com/articles will require the articles module and execute the nil function.

    You can create namespaces, by simply putting your controllers in sub-directories.

    For example, a request to mywebsite.com/blog/article/show/173, would require the article module from the controllers/blog folder, and execute the exported function show with 173 as parameter.

    Controller properties

    Inside controllers, you can access the standard request and response objects as this.req as this.res, respectively.

    For convenience, the following properties are also provided:

    this.COOKIES (cookies) this.SESS (cookie-based session) this.GET (query string) this.POST (request body)

    From REST to CRUD

    You can easily map REST requests to CRUD operations, by setting the REST property in the controller:

    exports.REST = true;

//GET
exports.read = function(id) {
  //Read
}

//POST
exports.create = function() {
  //Create
}

//PUT
exports.update = function(id) {
  //Update
}

//DELETE
exports.delete = function(id) {
  //Delete
}

    Views

    Rendering views is very easy. Inside a controller, execute:

    this.render(viewName, locals, callbak).

    If you omit the view name, it will be the same as your controller module. For example, let's suppose you have a controller module called hello:

    //File: app/controllers/hello.js

exports.nil = function() {
  //This will render the view 'app/views/hello.html'
  this.render();
}

    Models

    To access a model, you can use this.model(modelName) from any controller or service.

    Services

    To access a service, you can use this.service(serviceName) from any controller or service.
